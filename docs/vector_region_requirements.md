# ベクタ領域入力追加要件

## 1. 目的

現在は `region_id,x,y` 形式の CSV を使って抽出領域を定義している。  
追加機能として、領域定義入力に以下の GIS ベクタ形式を受け入れられるようにする。

- ESRI Shapefile (`.shp`)
- GeoPackage (`.gpkg`)

本書は、この追加機能の要件を整理するための補助ドキュメントである。  
既存の設計書本体には、実装完了時に反映する。

## 2. スコープ

今回の追加対象は、**領域定義入力の形式拡張** のみとする。

- 既存の入力点群テキスト処理は維持する
- 出力形式は維持する
- GUI / CLI の基本操作は維持する
- 既存の CSV 入力は引き続きサポートする

今回のスコープ外:

- 出力を GIS 形式で保存すること
- GUI 上で地図表示やプレビュー表示を行うこと
- 編集機能を持つこと
- CRS 変換の自動実行

## 3. 期待する利用イメージ

- 既存の CSV で領域定義する代わりに、SHP または GPKG のポリゴンレイヤを指定する
- 複数の領域定義ファイルを 1 回の実行で同時に指定する
- 1 フィーチャを 1 領域として扱う
- `region_id` はツールが自動生成する
- 出力ファイル名は既存ルールどおり `org_region{region_id}.txt` / `grd_region{region_id}.txt` とする

## 4. 機能要件

### 4.1 入力形式

領域定義入力として、以下を受け入れる。

- CSV
- SHP
- GPKG

ファイル拡張子で入力形式を判定する想定とする。
複数ファイルの同時指定を受け付ける。

### 4.2 ポリゴン読込

- SHP / GPKG から Polygon を読めること
- 各フィーチャを 1 領域として扱えること
- `region_id` は文字列として扱うこと
- `region_id` は利用者入力に依存せず、自動生成すること
- SHP では `入力ファイル名 + 連番` を基本ルールとする
- GPKG では `入力ファイル名 + レイヤ名 + 連番` を基本ルールとする

### 4.3 出力ファイル名

出力ファイル名は既存仕様を維持する。

- `org_region{region_id}.txt`
- `grd_region{region_id}.txt`

### 4.4 既存 CSV との互換

- CSV 入力は引き続き有効であること
- CSV / SHP / GPKG のいずれを使っても、領域表現の最終内部モデルは同じになること

## 5. データ制約

### 5.1 ジオメトリ

当面は以下を前提とする。

- Polygon のみサポート対象
- MultiPolygon は今回未対応とし、入力された場合はエラーとする

### 5.2 属性列

`region_id` は属性列から取得しない。  
利用者が `region_id` に相当する属性列名を指定する機能は今回持たない。

自動生成ルール:

- SHP: `入力ファイル名 + 連番`
- GPKG: `入力ファイル名 + レイヤ名 + 連番`

いずれも、内部では文字列として扱う。

### 5.3 GPKG レイヤ

GPKG は複数レイヤを持ちうるため、対象レイヤの決め方を定める。

- レイヤ指定がある場合は、そのレイヤを使う
- レイヤ指定が空白または未指定の場合は、読み取れた最初のレイヤを使う

最初のレイヤを使う場合、レイヤ順は読込ライブラリが返す順序に依存する。

### 5.4 座標系

現行要件に合わせて、**自動 CRS 変換は行わない**。  
入力点群と領域定義は同一座標系である前提とする。

ただし、SHP / GPKG は CRS 情報を持ちうるため、CRS 情報は読み取る。  
CRS が一致しない場合は **警告** とし、処理自体は継続する。

## 6. GUI / CLI 要件

### 6.1 GUI

- 領域定義ファイルとして CSV / SHP / GPKG を選択できること
- GPKG 利用時はレイヤ一覧を自動読込できること
- 読み取れたレイヤが 1 つだけなら、そのレイヤ名を表示すること
- 読み取れたレイヤが 2 つ以上なら、プルダウンで選択できること
- CSV 利用時には従来どおり追加指定は不要であること

### 6.2 CLI

- 領域定義ファイルの指定引数名は `--region-file` とする
- 既存の `--region-csv` は互換維持のため、当面は alias として残す
- GPKG 利用時はレイヤ名を任意で指定できること
- GPKG レイヤ名の指定引数名は `--region-layer` とする
- `--region-file` は複数回指定できること
- `--region-layer` は `filename=layer_name` 形式で複数回指定できること

## 7. エラー条件

少なくとも以下はエラーとして扱う。

- 領域ファイルが存在しない
- 拡張子は対象形式だが読込に失敗する
- Polygon 以外のジオメトリが含まれる
- `region_id` が重複し、出力ファイル名の一意性を壊す
- 同一座標系前提を満たさないと判断できるケース

## 8. 受け入れ条件

- CSV / SHP / GPKG のいずれでも領域定義を読み込める
- 複数の領域定義ファイルを混在して読み込める
- SHP / GPKG から `region_id` を自動生成して既存出力命名へ反映できる
- 既存の CSV 利用フローが壊れない
- GUI / CLI のどちらからでも対象形式を指定できる
- 不正なレイヤ指定や不正ジオメトリで適切に失敗する

## 9. 完了条件

- 必要ライブラリがプロジェクトへ追加されている
- Python 実装で CSV / SHP / GPKG の読み込みが統一的に扱える
- GUI / CLI の入力導線が更新されている
- テストが追加されている
- 実装完了後、`docs/requirements.md` と `docs/detailed_design.md` に反映されている

## 10. 現時点の推奨実装方針

- Python 側では `pyogrio` を入口にして GIS ベクタを読み込む
- ジオメトリ処理は `shapely` を使う
- CSV は現行ローダーを維持しつつ、最終的な内部 `Region` モデルへ揃える
