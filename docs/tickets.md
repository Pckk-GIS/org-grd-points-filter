# 31_point_filter チケット分解

## 前提

- 要件は [requirements.md](requirements.md) に準拠する。
- 詳細設計は [detailed_design.md](detailed_design.md) に準拠する。
- CLI を先に完成させる。
- CLI の単体テストと結合テストが安定して通った段階で GUI に着手する。

## 分割の考え方

- モデル、検証、読み込み、判定、書き込みを分離する。
- CLI は orchestration のみに寄せる。
- GUI は CLI 完了後に追加し、コアロジックを再利用する。
- 1 チケットで 1 つの責務だけを持たせる。

## 受け入れ条件

- 各チケットに完了条件があり、単独でレビュー可能である。
- CLI 完了前に GUI に着手しないことが明示されている。
- 実装順を見れば、どこから着手すべきか分かる。
