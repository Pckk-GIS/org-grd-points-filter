# 31_point_filter 要件定義

## 1. 目的

`input` 配下のテキストファイル群から、`xy.txt` 相当の領域定義に基づいて対象点を抽出する。

当初は矩形領域を想定していたが、実データは 4 点で定義された領域であることが分かったため、本要件では「4 点で囲まれる凸多角形」として扱う。
CSV は 1 行 = 1 頂点の点集合として管理し、同じ `region_id` に属する点群から自動で頂点順を決める。

## 2. 背景

- 入力データには `org` と `grd` の 2 系統がある。
- 入力ファイル群は系統ごとにまとめて処理する。
- 領域数は CSV に定義された `region_id` の数に依存する。
- 抽出結果は `region_id` ごとに `org` / `grd` それぞれ 1 ファイルずつ出力する。
- 領域定義ファイルは、現在の `xy.txt` は扱いづらいため CSV 化を検討する。

## 3. スコープ

### 3.1 対象

- `input` 配下のテキストファイル群
- 領域定義 CSV
- 抽出処理
- `region_id` ごとの出力ファイル生成
- CLI
- GUI
- PyInstaller による exe 化

### 3.2 対象外

- CRS 変換
- 座標系の自動推定
- 線形補間や座標補正
- 領域の再編集機能
- 凹多角形やより複雑な領域の正式サポート
- `vertex_order` を使った頂点順の明示管理

## 4. 現時点の理解

### 4.1 領域定義

- 現時点の領域は 4 点の座標で定義される。
- 現時点では凸多角形に限定する。
- CSV では 1 行 = 1 頂点として扱う。
- 同じ `region_id` に属する点群から、自動で頂点順を決める。
- ユーザーが CSV の行順を調整する必要はない。

### 4.2 入力ファイル

- `org` 系と `grd` 系を別々に扱う。
- 各入力ファイルは、どの列が `X`, `Y`, `Z` かを指定できる必要がある。
- 入力ファイルには列名がないため、列指定は列番号ベースとする。
- 必要のない列が含まれていてもよい。

### 4.3 出力

- 出力ファイル数は `region_id` 数 × 系統 2 つで決まる。
- 出力形式は入力と同じくテキストを基本とする。
- GUI からも同じ処理を実行できる。

## 5. 機能要件

### 5.1 領域定義ファイル

現時点では、`xy.txt` の代替として CSV 形式を採用する想定とする。

推奨フォーマット案:

```csv
region_id,x,y
1,-27777.482,-39671.252
1,-27821.133,-39782.979
1,-27091.157,-39981.623
1,-27129.094,-40072.025
```

要点:

- 1 行 = 1 頂点
- `region_id` で各領域を識別する
- `x`, `y` は必須
- 頂点数は固定しない
- 行順は自由でよい
- 同じ `region_id` の点群から凸包を自動生成する

### 5.2 入力列マッピング

入力テキストごとに、以下を指定できること。

- `X` 列
- `Y` 列
- `Z` 列

列指定の前提:

- 列番号指定のみとする
- 列番号は 1 始まりとする
- ヘッダ行はない前提とする

### 5.3 抽出条件

- 各点がどの領域に属するかを判定し、該当する行を抽出する。
- 各領域の矩形範囲で事前判定し、明らかに外側の点は多角形判定から除外する。
- 入力はファイル単位で並列処理し、各ファイルは逐次読み込みながら抽出する。
- 境界上の点は含める。
- 1 点が複数領域に重なる場合は、該当するすべての領域へ出力する。

### 5.4 出力ファイル

- 系統ごとに `region_id` 数ぶんのファイルを出力する。
- 出力ファイルは抽出対象のみを含む。
- 出力ファイル名は `org_region{region_id}.txt`、`grd_region{region_id}.txt` とする。
- 出力はメモリに全件保持せず、逐次書き込みで処理する。
- 成功時は一時ファイルから本番ファイルへ置き換える。
- 同じ `region_id` に属する点群から凸包を作れない場合はエラー終了する。
- 面積は補助的な確認材料とし、面積だけでは正常性を判定しない。

### 5.5 GUI と exe 化

- GUI は CLI と同じコアロジックを利用する。
- GUI は PyInstaller で exe 化できることを前提とする。
- まずは `onedir` 形式でのビルドを想定する。
- GUI は入力パス、領域 CSV、列番号、出力先を画面から指定できること。
- GUI にはメニューバーを用意し、`ヘルプ` から使い方を開けること。
- ヘルプ内容は、単体でツールを渡しても使える程度に詳細であること。
- GUI 用の起動入口はリポジトリルートに配置し、PyInstaller はその入口を参照すること。
- PyInstaller 用の spec ファイルをリポジトリで管理する。

## 6. 制約・前提

- CRS 変換は行わない。
- 入力ファイルと領域定義は同一座標系であることを前提とする。
- `EPSG:6677` のような情報があっても、現時点ではメタ情報として保持するのみで、変換処理には使わない。
